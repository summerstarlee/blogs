# 设计模式

## 分类

1. 创建型模式: 提供创建对象的机制， 增加已有代码的灵活性和可复用性
2. 结构型模式: 将对象和类组装成较大的结构，并同时保持结构的灵活和高效
3. 行为型模式: 负责对象间的高效沟通和责任委派

迭代器模式
 要想成为一个可迭代的对象， 其内部必须实现 `@@iterator` 方法。 该方法可以通过 `Symbol.iterator` 属性访问

 ```js
 var a = []
 console.log(a)
/**
 length: 0
 [[Prototype]]:
    Symbol.iterator: f values()
 * */
 ```
